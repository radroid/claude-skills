## Journey: <name — e.g. "First-time user sends an invoice">

**Trigger:** what just happened that puts the user at the start of this journey
(e.g. "user clicked the email signup confirmation link")

**Preconditions:** what must already be true
(e.g. "user has confirmed email; no clients exist yet; no invoices exist yet")

**Success outcome:** the state the user is in when done
(e.g. "an invoice is created, marked sent, and the user is on its detail page")

### Happy path

1. Lands on `<slug>.html` → <user action>
2. → navigates to `<slug>.html`, <what they do here>

4. <…until success outcome>

### Branches & failures

- **<precondition violated>** → at step `<N>`, <what happens instead>
  (e.g. "**No clients exist** → at step 2, 'New invoice' CTA opens client-create modal first")
- **<validation failure>** → at step `<N>`, <what happens instead>
  (e.g. "**Form invalid** → stay on `invoice-new.html`, inline field errors")
- **<external dependency fails>** → at step `<N>`, <what happens instead>
  (e.g. "**Email send fails** → stay on `invoice-detail.html`, error banner, 'Retry' button")
